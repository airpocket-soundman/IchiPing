C4. IchiPing — イチ発のピンで、家のすべての窓を聴く

マイク 1 個 + NN で、窓 3 + 扉 2 + 連続音響空間トポロジを 同時に 推定する卓上 PoC エッジ AI
DigiKey M1 ROHM EDGE HACK 本命: 音響+NN 法 補助: 気圧スパイク法

0. このページの位置づけ

原案 C4 DoorBaro（[一覧 §C4]）は気圧 1 個でドア開閉・換気・HVAC・天気を同時推定するアイデアだった。物理的に成立はするが、「外出先から、いま窓が開いているか確かめたい」という具体ユースケースには気圧スパイク法では即応性・状態性で不足する（スパイク = イベント検出であって、状態のスナップショットを取りにくい）。本プロジェクト IchiPing はこの原案からピボットした名称である。

本ページでは 気圧法を補助として残しつつ、本命を「リファレンストーン放射 + NN による開窓状態分類」に置き換える方針を整理する。

1. ユースケース — 「雨が降ってきた、窓は閉めてきた？」

外出中に天気が崩れたとき、ユーザーがスマホから問い合わせると、家に設置された 1 台のデバイスが 「いま家のどこかの窓が開いているか／全て閉まっているか」 をリアルタイム判定して返す。

1.1 想定シナリオ

  • 通勤中に雨予報を見て不安になる → スマホアプリで 窓チェック をタップ
  • デバイスがリファレンストーンを 3〜5 秒放射 → マイクで取得 → NN 判定 → スマホに結果プッシュ
  • 「連続空間内のいずれかの窓が開いています」 or 「全て閉まっています」を即返答
  • 定期測定（30 分毎など）で状態変化検出 → プッシュ通知（"3 分前まで閉まっていた窓が開きました"）

1.2 なぜ既存ソリューションでは足りないか

  • 窓ごとの磁気スイッチ: 各窓に取り付ける必要があり、賃貸・既設窓には不向き。設置コスト大
  • カメラ監視: プライバシー強で住人受けが悪い。死角もある
  • 気圧スパイク（原案）: 開閉イベントは取れるが、出先から「いま開いているか」を尋ねる用途では履歴推定に頼ることになり、起動以降のイベント漏れに弱い

本案の強み: 1 部屋に 1 個置くだけ、能動的問い合わせに即応、プライバシー完璧（音だけ）、賃貸でも使える。

2. 検出スコープ — 「連続音響空間」

本デバイスが検出できる範囲は、設置部屋から開いた扉でつながっている全部屋の窓。部屋間扉が閉まっていればその先は不可視。

2.1 連続音響空間の概念図

2.2 物理的根拠

  • 音響的に連続な空間は 共有された残響モードを持つ。Sabine 式 T60 = 0.16·V/A で、容積 V と吸音面積 A は連続空間全体の合算
  • 開窓 = 開口面で吸音率 ≈ 1.0 → 連続空間の総吸音 A が増加 → RT60 が短くなる
  • 同時に 外部音の浸入（道路ノイズ、風切り音）→ ノイズ床スペクトルが変化
  • 低域モード（部屋共振）の 共鳴 Q 値が低下 → 周波数応答のディップ
  • 部屋間扉が閉まっていれば、扉の透過損失（一般的に 20〜30 dB）でその先の音響変化はほぼ届かない → 検出範囲が「連続空間」に自然に限定される
仕様の正直さ: 「家全体の窓を 1 個で見る」ではなく 「設置部屋から音響的に到達できる範囲の窓を見る」。ユーザーには UI 上で「外出時は寝室の扉を開けておいてください」程度のガイダンスを提示する設計にする。

3. 信号処理パイプライン

3.1 全体フロー

励振: MCXN947 I²S → MAX98357A → スピーカ。リファレンストーン放射（chirp 200 Hz〜8 kHz スイープ 300 ms、または擬似乱数ノイズ 1 s）
▼
受音: I²S MEMS マイク（SPH0645LM4H）で 16〜48 kHz 取得、DMA でリングバッファへ
▼
RIR 推定: PowerQuad DSP で 1024 点 FFT → 励振信号で割って室内インパルス応答（RIR）抽出。約 1 秒分
▼
特徴抽出: バンド別 RT60、DRR（直接波/残響比）、低域モードピーク、ノイズ床スペクトル
▼
NN 推論: eIQ Neutron NPU 上の 1D CNN（〜200 K params, INT8）で "連続空間内に開窓あり / 全閉" の二値分類（v2 では開口面積回帰）
▼
通信: WiFi/BLE モジュール（例: ESP32 を補助 MCU として）でクラウド or 直接スマホへ結果送信

3.2 なぜ NN が不可欠か

  • RIR 特徴は 家具・人の存在・室温・湿度・時間帯（外部騒音レベル）で大きく変動する
  • 「窓が開いた瞬間にどれかの特徴量が単調に動く」というしきい値モデルは現場で破綻する
  • 分布学習による "その家での全閉時の RIR 分布" からの逸脱検出（OOD）が本質
  • v1 は「全閉 vs それ以外」の二値分類 → autoencoder + 再構成誤差 or 一クラス分類で実装。校正データの大半が「全閉」だけで済むのが大きな実用利点

4. NN モデル設計

入力RIR の最初の 0.5〜1.0 秒（500〜1000 サンプル @ 1 kHz リサンプル）or バンド別特徴ベクトル（オクターブバンド × RT60/DRR/ピーク = ~30 次元）
v1 出力二値: 全閉 / いずれか開 + 信頼度
v2 出力多クラス: 全閉 / 1箇所小開 / 1箇所大開 / 複数開 + おおまかな方角分類（多マイク化）
アーキテクチャ v11D conv autoencoder（〜100 K params, INT8）。再構成誤差を OOD スコアとする
アーキテクチャ v21D CNN + 全結合分類ヘッド（〜300 K params）。マルチタスク学習で開口面積回帰も
学習データ家ごとに校正期間が必要。設置直後にユーザー操作で「全閉」を 50〜100 回（時間帯を散らして）録音 → autoencoder 訓練。任意で「窓1だけ開」「窓2だけ開」も録音すれば v2 へ拡張可
転移学習image-method の室内音響シミュレータ（pyroomacoustics 等）で家ごとに合成データを生成し、現場校正は短縮する

4.2 DigiKey v1 と ROHM v2 の NN 設計思想の違い

同じ実験模型・同じ学習データを使うが、ターゲット MCU の制約が NN アーキテクチャを根本的に分岐させる。両者は性能 vs 適応性の異なる軸を最適化する設計になる。

DigiKey v1: 深層 NN ＋ オフライン学習

  • ハードウェア: MCXN947 + eIQ Neutron NPU + PowerQuad DSP
  • 制約: モデルサイズ ~500 KB INT8 まで、隠れ層数自由
  • アーキテクチャ: 多層 1D-CNN（3〜5 層）+ マルチタスク出力ヘッド ── 単一ネットワークで全タスク同時推論
  • 学習: PC 上で backprop + Adam による標準訓練、INT8 量子化してデプロイ
  • 特徴量: 生 RIR or 周波数応答を直接入力 → NN が特徴学習
  • パラメータ規模: 〜30 K params, ~30 KB INT8
入力: 周波数応答スペクトル (1024 bin)
  │
  ├─→ Conv1D 16ch k=16 s=4 + ReLU
  ├─→ Conv1D 32ch k=8  s=4 + ReLU
  ├─→ Conv1D 64ch k=4  s=2 + ReLU
  ├─→ Global Average Pool
  │
  ├─→ FC head_1 ─→ any_open (binary)
  ├─→ FC head_2 ─→ door_AB (continuous)
  ├─→ FC head_3 ─→ door_BC (3-class incl. 不可視)
  ├─→ FC head_4 ─→ window_a (continuous)
  ├─→ FC head_5 ─→ window_b (3-class)
  └─→ FC head_6 ─→ window_c (3-class)
        ↑
   全ヘッドが 共有 backbone を活用 ── マルチタスク学習で精度向上

ROHM v2: ELM ＋ 自律オンデバイス学習

  • ハードウェア: ML63Q2557（Cortex-M0+ @ 48 MHz）+ AxlCORE-ODL ハード AI アクセラレータ
  • 制約: 3 層 FFNN（隠れ層 1 層のみ）、隠れ層重み α はランダム固定、出力層 β のみ学習可
  • アーキテクチャ: 専門 ELM（二値分類）×4 スロット + 階層推論 + マハラノビス距離（補完）
  • 学習: 閉形式（疑似逆行列）でデバイス上完結。最小二乗で β を一発算出 ── PC 不要
  • 特徴量: 物理特徴量の人手設計が必須（RT60 per band、DRR、モードピーク位置等）── ELM は特徴を学べない
  • パラメータ規模: 1 スロット 〜4 KB INT8 × 4 スロット = ~16 KB
入力: 物理特徴量 (~30 次元 / RT60×8帯域 + DRR + モードピーク等)
  │
  │ [マハラノビス層 (M0+ で実行)]
  ├─→ d²_k for k ∈ 14 状態 → argmin → 粗いトポロジ判定
  │
  │ [ELM 専門モデル群 (AxlCORE-ODL で実行)]
  ├─→ スロット 1 ELM-AB ── 常時 ── door_AB 二値
  ├─→ スロット 2 ELM-a  ── 常時 ── window_a 二値
  ├─→ スロット 3 ELM-BC ── AB=開時のみ ── door_BC 二値
  └─→ スロット 4 ELM-b/c ── 時分割切替 ── window_b/c 二値
        ↑
   各 ELM は 独立したランダム射影 + 線形読み出し
   観測可能性の階層でモデル起動を制御

本質的な違い ── 性能 vs 適応性のトレードオフ

DigiKey v1 = 「訓練を済ませた最強の NN を載せる」
  • 実験模型で集めた 10,000 サンプルを backprop で深く学習
  • マルチタスク共有 backbone で全タスクを同時学習
  • NPU のフル性能を活かして高精度を狙う
  • 環境変化への対応は PC 経由の再学習 → デプロイ
  • 「ベンチマーク勝負」モデル
ROHM v2 = 「制約を逆手に取り、現場で学ぶ」
  • ELM の閉形式学習で「いま設置した家」専用モデルをその場で訓練
  • 新しい家・新しい家具配置に 30 秒で再校正できる
  • NN の表現力は犠牲にするが、適応性で取り返す
  • "自律的に学ぶエッジ AI" という ROHM コンテストテーマと完全一致
  • 「フィールド勝負」モデル

同じ実験データの二様の使い方

§7 の実験模型で集めた 1,000〜10,000 サンプルは 両系統で使うが消費の仕方が異なる:

用途DigiKey v1ROHM v2
主タスク学習backprop で全ヘッド同時最適化ELM-AE: 「全閉」サンプルのみで β を最小二乗
補助タスク学習同上各 ELM 専門モデルを個別に最小二乗で訓練
マハラノビス用統計使わない14 状態クラスの μ_k, Σ_k を集計
検証分割未経験状態をホールドアウト同左 + 「家を変えた仮想シナリオ」も評価
転移学習新しい家ごとに backprop で fine-tune新しい家での「全閉」録音 30 秒で β 再計算

実験データの収集設計は v1 / v2 のどちらにも対応できる物理特徴抽出パイプラインを通すことで両系統に共通インフラを提供する。生 RIR と物理特徴量の両方を保存しておけば、後から DigiKey 用に深層 NN、ROHM 用に ELM の双方を学習できる。

4.4 ROHM v2 NN 詳細設計

マハラノビス距離（M0+ で実行）と専門 ELM × 4（AxlCORE-ODL で実行）の並列構成。階層推論で必要な ELM のみ起動する。

マハラノビス層スペック

入力次元26（物理特徴量）
クラス数14（観測可能性で縮約した topology × 開閉組合せ）
パラメータ14 × (μ_k 26次 + Σ_k 26×26) ≈ 9.8 K（float32）
サイズ~10 KB（フル QDA）／ ~3 KB（LDA で共有 Σ）／ ~4.5 KB（対角 Σ）
推論14 × 26² ≈ 9.5 K mul + 14 比較 → ~0.5 ms on M0+ @ 48 MHz

ELM × 4 スロット スペック

スロットタスクD / N / Kパラメータ動作条件
1ELM-AB (door_AB 二値)26 / 64 / 1α 固定 1,664 + β 学習 64常時動作
2ELM-a (window_a 二値)26 / 64 / 1同上常時動作
3ELM-BC (door_BC 二値)26 / 64 / 1同上AB = 開のときのみ
4ELM-b/c (window_b/c 二値, 時分割)26 / 64 / 1α 共通 + β 2 種観測可能時のみ
合計~7 KB INT8

α（隠れ層重み）はランダム初期化のまま固定で Flash に保存。β（出力層重み）のみ最小二乗で学習し、RAM に保持して再校正に対応。

マハラノビス統計と ELM β のデータ源

μ_k, Σ_k, β はすべて 同じ実験模型から得た学習データ（§7.4 のプロトコルで収集）から計算される。v1 と v2 はデータを共有し、消費の仕方だけ異なる。

[共通: 学習データ収集]
   実験模型 (§7) → 1,000〜10,000 サンプル (生音声 + サーボ角度ラベル)
        │
        ▼
   PC で物理特徴抽出: 各サンプル → (生音声, 26次元特徴, ラベル) 三重組

[v1 系統 (PC でオフライン学習)]
   生音声 (or 周波数応答) → CNN backprop 訓練 → INT8 量子化 → MCXN947 デプロイ

[v2 系統]
   μ_k 計算: 各クラス k (=1..14) に属するサンプルの特徴量平均
   Σ_k 計算: 各クラスの特徴共分散行列（Ledoit-Wolf 縮小推定で安定化）
   β-AB 計算: 全サンプルから最小二乗 β = H†T
   β-a  計算: 同上
   β-BC 計算: AB=開のサンプルのみで最小二乗
   β-b, β-c 計算: 観測可能サンプルのみで最小二乗
        │
        ▼
   {μ_k, Σ_k, α (固定 seed), β_*} を ML63Q2557 Flash に書込

オンデバイス再校正（Solist-AI のキラーデモ）

v2 の固有価値は 新環境で 30 秒の再校正ができること。デモの見せ場:

  1. 制御ボックスの「再校正」モードスイッチを ON
  2. ML63Q2557 がチャープ放射＋マイク取得 を 30 回連続実行（≈ 30 秒）
  3. 取得した特徴量の平均を新しい μ_全閉 として更新、Σ_全閉 も再計算
  4. ELM の β は前のまま流用、または α 固定なので β だけ再学習も選択可
  5. 30 秒で完全自律的に校正完了 → 通常動作に復帰

デモ動画では「センサ位置をずらして再校正ボタンを押し、30 秒待つだけで動作復帰」を実演。クラウド・PC なしで完結する "自律学習エッジ AI" を象徴するシーン。

5. 気圧ヘッド（補助センサ）の役割

音響リファレンストーン法

→ 状態のスナップショット

  • 外出先からの能動的問い合わせに即応
  • 「いま開いているか」を直接判定
  • 校正後は同じ環境で動作
  • 音を出す = 在室時はうるさい（夜間は無音モード）

気圧スパイク法（補助）

→ イベントの即時検出

  • 窓開閉・ドア開閉の瞬間に圧スパイク
  • 無音・常時動作（電池駆動可）
  • HVAC・換気扇起動も検出
  • "いま" が分からない（履歴推定が必要）

統合戦略: 気圧ヘッドが「窓を開けたっぽい」イベントを検出 → 音響法を自動起動して状態確認 → ユーザーに通知。能動 + 受動の二段構え。

6.1 ストレージの選定理由

学習データ収集（§7.4）で生音声を保存する手段として、Ethernet + サーバ転送ではなく microSD（SPI）+ 装置内ローカル保存を選択する理由:

選択肢メリットデメリット判定
microSD（SPI） 配線 4 本、FatFs 標準ライブラリ、64-256 GB 容量、$10 で完了 後でカード抜き差し必要 ◎ 採用
Ethernet（PHY + RJ45） リアルタイム転送、複数装置の集約 ネットワーク認証・設定が煩雑、PHY + 50MHz xtal + RJ45 で部品増、lwIP 実装重い ✗
Wi-Fi（ESP32 経由） 無線でフレキシブル SSID/パスワード設定の UX が必要 △（無線本来の用途で別用）
USB-C CDC（PC 直結） 追加ハードゼロ PC 必須、模型と物理的に近接必要 ○（v1 PoC では併用）

結論: ネットワーク認証の手間を避け、microSD を SPI 接続で内蔵。1.5 s/サンプル × 10,000 = 4 時間連続収集で 650 MB、64 GB カードなら100 回分の収集セッションに対応。データ転送は PC に挿し直して読む方式で十分速い（USB 3.0 リーダで 1 GB/分以上）。

7. 実験模型と学習データ収集計画

サーボで駆動する 3 部屋模型を作り、扉と窓をランダムに動かしながら音響計測することで、大量の学習データを自動・自己ラベル付きで取得する。本案の最大の弱点である「家ごとの校正コスト」をシミュレータで解決する設計でもある。

7.1 模型の構成

配置ポイント:

  • 扉 AB: Room A と Room B を仕切る 150 mm 壁の 左隅に設置（100×50 mm, 高さ×幅）
  • 扉 BC: Room B と Room C を仕切る 150 mm 壁の 右隅に設置 — AB とは対角配置
  • 窓 a, b, c: 各部屋の左短辺 100×100 mm 壁の中央に設置（50×50 mm）
  • SP/MIC: Room A の 右短辺壁（窓 a の対面）に設置
  • 扉のジグザグ配置により、ABC 連結時の音響経路が直線でなく曲がりを持ち、モード構造がより豊かになる

7.2 5 個のサーボと観測可能性

扉 AB / BC の開閉により、Room A から見た連続音響空間は 3 つのトポロジ状態に縮約される。それぞれの状態で どのサーボの状態が音響から推論できるか（観測可能か） は次の通り。扉自体の状態も観測可能性が変動する点に注意:

扉 AB扉 BCA から見える連続空間観測可能（推論できる）観測不可（推論できない）
閉—{ A }a, ABb, c, BC
開閉{ A, B }a, b, AB, BCc
開開{ A, B, C }a, b, c, AB, BC—

観測不可性は窓だけでなく扉自体にも及ぶ:

  • AB が閉 → B 側の世界全体（窓 b, 窓 c, 扉 BC のいずれの状態も）A の音響では一切区別できない
  • BC が閉（AB は開） → 窓 c の状態は A から推論できない
  • これは情報理論的な限界であり NN の表現力では超えられない。ラベル設計で明示的に「不可視」を教えるのが鍵

7.3 ラベル設計 — マルチタスク + 不可視クラス

サーボ角度から自動で取れる教師信号を主タスク・補助タスクとして並べる。観測不可なゲート（窓 b, c および扉 BC）には「不可視」という第三クラスを明示的に教えることで、NN が物理的に推定できないものを推測しようとして混乱するのを防ぐ。

ラベル種類タスク役割
any_open_in_connected_space二値主タスク（外出時通知の判定そのもの）
door_AB連続 [0, 1]（常に可視）補助・トポロジ復元の根幹
door_BC三値（開 / 閉 / 不可視）補助・AB 閉時は不可視
window_a連続 [0, 1]（常に可視）補助・回帰
window_b三値（開 / 閉 / 不可視）補助・AB 閉時は不可視
window_c三値（開 / 閉 / 不可視）補助・AB 閉 or BC 閉時は不可視
n_open_visible整数 0〜3補助・回帰
total_open_aperture_cm²連続（可視部分のみ合算）補助・物理的に最も意味あり
connected_volume_L連続補助・Sabine 式の検証用

マルチタスクで学習すると、限られたデータでも 共有表現 が物理的に意味ある特徴に収束しやすい。主タスク単独学習より汎化が良くなる典型例。

7.4 データ収集プロトコル

1 サンプル ≈ 3〜5 秒:

  1. サーボにランダム角度コマンド送信
  2. 動作完了 + 気流安定で 2〜3 秒待機
  3. chirp（200 Hz〜8 kHz スイープ）を 1 秒放射
  4. 残響を含めて 1 秒取得
  5. サーボ角度を読み出し、音響データと一緒に CSV / npz で保存

サンプリング戦略:

  • 層化サンプリング: 32 状態（2 扉 × 3 窓のオンオフ）が均等に出るよう乱数生成。ナイーブな uniform は確率分布が偏る
  • 連続角度の組み込み: 中間角（半開）も入れて回帰タスクを学習可能に
  • 時間帯散らし: 1 日連続稼働で朝・昼・夜の周囲ノイズ分布を学習
  • 外部ノイズ多様化: 完全防音だと実家屋への転移性が落ちる。実験中に意図的に音楽／会話を流す、または augmentation でホワイト・ピンクノイズを加算

総サンプル数目安: 1,000〜3,000。3 秒/サンプルなら 1.5〜4 時間連続稼働で揃う。一晩動かせば 1 万サンプル超。

7.7 検証分割

ランダム split は精度を過大評価する（隣接状態が train / test に分散するため）。推奨:

  • 未経験組み合わせを完全ホールドアウト: 例えば「AB=70°, BC=30°, a=open, b=closed, c=open」のような特定状態を学習から除外して評価
  • 時間ホールドアウト: 後半の時間帯（=異なるノイズ条件）を test に回す
  • 形状摂動: 別ロットの実験で窓の形状（正方形 vs 円形 / 開口面積を変える）を変更し、学習した特徴が物理本質か模型固有かを切り分ける

7.8 実験模型固有の落とし穴

  • 気密性不足: 段ボール扉の閉じ込めがゆるいと "閉" 状態でも空気漏れがあり信号が歪む。シール材で実家屋に近い気密性を確保
  • スケール効果: 30 cm 部屋では音響モード周波数が実家屋の ~10 倍にシフト → chirp 帯域を 1〜20 kHz に上げて等価な現象を引き出す
  • サーボ動作音の混入: chirp 放射前にサーボ動作を完了 + 十分待機。サーボノイズ自体を別ヘッドで識別させて除外する手も
  • 過学習による模型記憶: NN が "この模型固有の部屋音響" を記憶してしまう → 形状摂動セッションで切り分け、必要なら augmentation 強化
  • 不可視ラベルのバランス: AB 閉のとき 窓 b, c に加えて扉 BC も不可視になり（AB 閉サンプル ≈ 半数）、補助タスク 3 つが同時に "不可視" タグを抱える。BC 閉のときはさらに窓 c も不可視。クラス重みづけ＋カリキュラム学習（全開→中間→全閉の順）で平準化

  • サンプリング多様性: セッション間で時間帯・室温（必要なら）・外部ノイズ条件を変える
  • セッションメタデータ: 開始/終了時刻、室温、収集モード（層化／ランダム）等を HDF5 attributes として記録
  • 検証用サンプル: 各セッションの最後の 10% は "未経験組み合わせ" を意図的に踏む（§7.7 検証分割用）

8.1 デモ用の模型

9. 「イチ」物語化（DigiKey 応募軸）

DigiKey の "イチ" は センサ数 vs 監視対象数の比率に振る。1 個のセンサが複数の対象を同時把握するという 「1 対多」の劇的さ が、応募ページ・2 分動画・SNS で最も伝わる軸。

「イチ個のセンサで、複数のモノの状態を把握する」

  • 定量: 「マイク 1 個 から、窓 3 + 扉 2 + 連続空間トポロジ = 計 6 種類の対象を同時推定」
  • 比率: 「センサ 1 ÷ 監視対象 6 = 1 / 6 — エッジ AI の "最小化" を最も先鋭化した形」
  • 体験: 「外出先からも、家のイチ瞬を確かめる」
  • 製品: 「イチ台に集約された "外気接触" 検知デバイス」

"1 対多" を強調することで NN の存在理由も自然に立つ: 単純な信号処理では 1 対 1 の検出しかできない（マイク 1 個では音量しか分からない）が、NN によって 1 つの入力ストリームから複数の状態を分離できる。これは NN 必然性そのもののストーリー化でもある。

10. Solist-AI 移植時の特殊課題（ROHM v2 実証スコープ）

v2 は「Solist-AI の厳しい制約下でも v1 の自由な NN に近い結果が出せること」の技術実証に絞る。実家屋デプロイは本プロジェクトのスコープ外。下記は v2 デモで検証すべき課題:

  • ELM 1 隠れ層の表現力限界 → 専門 ELM ×4 アンサンブル + マハラノビス併用で多クラス分類精度を v1 の 5% 差以内に詰められるかを実証
  • α 固定（ランダム）のばらつき → α の乱数 seed を変えて 10 回学習し精度分散を測定。ブートストラップアンサンブルで吸収できるかを検証
  • 4 スロット制約での時分割切替レイテンシ → window_b と window_c を時分割で切替えるとき、推論あたりの追加遅延を計測（数 ms 想定、UX 上問題ないか）
  • 不可視ラベルのクラス不均衡 → AB 閉時のサンプルが多くなり β の最小二乗解が偏る。クラス重み付き擬似逆行列の効果を実証
  • マハラノビス距離計算の M0+ 上での実装コスト → 14 クラス × D=30 次元の d²_k を 1 ms 以内に計算できるか確認
  • オンデバイス学習のデモシナリオ → 実験模型上で「センサを物理的に移動 → 30 秒で再校正 → 動作復帰」を示す。Solist-AI の自律学習機能のアピール

11. v1 と v2 の関係 — 同じ問題を 2 つの NN 制約で解く

v1 → v2 は機能拡張ではなく 同じ問題（同じ実験模型、同じ訓練データ、同じ推論ターゲット）を、ハード制約の異なる 2 つの NN プラットフォームで解く並列実装。両者の精度差・コード差・アプローチ差そのものが作品の成果。

段階NN プラットフォーム推論ターゲット主張
v1（DigiKey, 〜6/22） MCXN947 + eIQ Neutron NPU
多層 1D-CNN（自由設計）		 主タスク二値 + 補助マルチタスク（door/window 状態 + 角度回帰） 自由な NN で何がどこまでできるかのリファレンス精度を確立
v2（ROHM, 〜9/28） ML63Q2557 + AxlCORE-ODL
ELM（1 隠れ層・α 固定）×4 + マハラノビス		 同上（同じ推論ターゲット） 強い制約下でも v1 に近い精度が出せることを実証

実家屋デプロイ・雨センサ連携・侵入検知などの拡張は 本プロジェクトのスコープ外。両系統とも実験模型（§7）上でのみ評価する。

12. 次にやること

v1（DigiKey 締切 6/22）と v2（ROHM 締切 9/28）の 2 フェーズに分けて整理。実家屋デプロイはスコープ外。

Phase 1: 実験模型と v1（DigiKey 締切まで）

  • 透明アクリル 5 mm の 3 部屋模型（150×100×100 mm 各室）試作、扉対角・窓中央のサーボ駆動機構（§7.1 図）の組み立て
  • PCA9685 + SG90 × 5 の PC 制御テスト（USB シリアル → I²C 経由で 5ch PWM）
  • PC オーケストレータ Python スクリプト: ランダム角度生成 → サーボ駆動 → 待機 → MCXN947 への計測トリガ → データ保存（CSV / npz）
  • 層化サンプリング実装（観測可能性で縮約した 14 状態 + 中間角を均等に踏むスケジューラ）
  • pyroomacoustics で組合せ RIR シミュレーション、物理特徴量（RT60, DRR, モードピーク）の分離度確認
  • MAX98357A + SPH0645LM4H + MCXN947 の I²S 結線テスト、PowerQuad での FFT・周波数応答抽出ベンチマーク
  • v1 マルチタスク 1D-CNN（主二値 + 補助 6 出力）の PC 上での backprop 学習、INT8 量子化 → MCXN947 デプロイ
  • 1,000 サンプル収集 → v1 学習 → 評価のループを 1 サイクル完走
  • 2 分デモ動画撮影、ProtoPedia ページ作成、myLists 部品表登録

Phase 2: Solist-AI 移植実証（ROHM 締切まで）

  • Phase 1 で収集した実験データを Solist-AI 用に再構成（生 RIR → 物理特徴量 30 次元へ）
  • 専門 ELM × 4（ELM-AB / ELM-a / ELM-BC / ELM-b・c 時分割）の β を Solist-AI 上で閉形式学習、AxlCORE-ODL の動作確認
  • マハラノビス距離（14 クラス分の μ_k, Σ_k）の Cortex-M0+ 上での実装、推論 1 ms 以内を確認
  • 階層推論パイプライン: ELM-AB → トポロジ判定 → 関連 ELM 起動の制御コード
  • v1（自由 NN）と v2（ELM）の精度比較: 同じ評価データで離散分類・連続回帰の精度差を測定
  • 自律学習デモ: センサ位置を変えて 30 秒で β 再計算 → 動作復帰を実機実演
  • 「Solist-AI 制約下でも v1 から 5% 以内」を達成できるかの検証 → 達成できなければアーキ調整
