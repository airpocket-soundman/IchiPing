IchiPing firmware

MCUXpresso 用の最小スケルトン。まだ実ハード（INMP441 マイク / MAX98357A / PCA9685 など）には触らず、 合成 chirp + 残響テールを生成して OpenSDA デバッグ UART に流すだけ。 PC 側 pc/receiver.py と組み合わせて、シリアル経路と保存パイプラインを先に通すための足場。
v0.1 現フェーズ FRDM-MCXN947 ターゲット MCUXpresso 11.9+ 必要

構成

firmware/
├── include/
│   ├── ichiping_frame.h   フレームフォーマット定義（PC 側と共有する単一の真実）
│   └── dummy_audio.h
├── source/
│   ├── main.c             SysTick + LPUART + フレーム送信ループ
│   ├── ichiping_frame.c   ヘッダ・ペイロード・CRC-16/CCITT パッカー
│   └── dummy_audio.c      合成 chirp 200→8 kHz + xorshift32 ホワイトノイズ残響
└── README.md / README.html

ビルド手順（MCUXpresso IDE）

  1. MCUXpresso SDK for FRDM-MCXN947 をインストール（11.9 以降）
  2. File > New > Create a new C/C++ Project → frdmmcxn947 → 例として driver_examples > lpuart > polling_transfer をベースに作成
  3. プロジェクト名を ichiping_dummy などに変更
  4. デフォルトの main.c を source/main.c で置き換え
  5. プロジェクトに以下のソースを追加:
    • source/ichiping_frame.c
    • source/dummy_audio.c
  6. Properties > C/C++ Build > Settings > Includes に ${ProjDirPath}/../firmware/include を追加（あるいはヘッダを source/ 直下にコピー）
  7. ビルドして OpenSDA で書き込み → 緑の PWR LED 想定箇所 (Arduino A0) は未配線でも問題なし

シリアル設定

ボードFRDM-MCXN947、OpenSDA デバッグ COM ポート
ボーレート921600 bps（PC 側もこの値）
フレーミング8N1、フロー制御なし
注意バイナリ転送のため 端末ソフトでは開かないこと（pc/receiver.py で受ける）

動作シーケンス

フレーム形式

ichiping_frame.h が単一の真実（single source of truth）。PC 側は pc/ichp_frame.py の HEADER_FMT を同期させる。 ドリフトすると CRC が通らず無音状態になるため、ヘッダ変更時は必ず両方を更新し python -m unittest pc.test_frame_format -v で 9 テスト全パスを確認。
