firmware/shared/include/servo_driver.h がビルド時シンボルでバックエンドを切り替える共通 API: SERVO_BACKEND_PCA9685（16 ch, NXP, 既定）/ SERVO_BACKEND_LU9685_I2C（20 ch, 中国製互換）。 pca9685.c は PWM tick + SG90 校正、lu9685.c は ESPEasy 由来プロトコル（reg 0xFC=freq, 0xFD=20ch bulk, reg=pin で個別）。 main_servo_test.c は共通 API のみ呼び、ハード差し替え時のコード変更ゼロ（Preprocessor シンボルを変えるだけ）。同期ウェイポイント (0/90/180°) → 5 ch ソロスイープのループ。LU9685 実機動作確認済（commit b22a9f0、現物 I²C アドレス 0x1F、i2c_scan() 診断付き）。

担当: MCU + ハード 済: LU9685 5 ch スイープ + アドレス確定 (0x1F) 未着手: PCA9685 実機キャリブ
