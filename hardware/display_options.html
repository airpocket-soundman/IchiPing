ディスプレイ候補比較

spec.html §6 の SSD1306 0.96" から進化版を選び直す。 用途は「5 部屋の窓・扉状態 + 推論結果 + 信頼度 + EXEC モード表示」。 視認性・配線本数・MCU 負荷・デモ映えの 4 軸で比較し、本リポで採用する 1 つを決める。
5 候補比較 ILI9341 2.4" 採用決定 SPI microSD と共有

用途と要件

表示すべき情報 ・5 部屋の窓/扉状態（窓 a/b/c + 扉 AB/BC）
・推論結果（クラス + 信頼度）
・EXEC モード状態（収集中 / 待機 / エラー）
・seq カウンタ + ハートビート
必要解像度128×64 で最小限。128×128 / 240×135 で 状態のアイコン化がグッと改善
応答時間0.5 秒以内で 1 画面更新（I²C 400 kHz では 128×64 = 1 KB を ~20 ms）
視認性3 m 離れたデモ動画で読める ≒ 1.3" 以上を推奨
バス制約FC4 I²C は PCA9685 0x40 / BMP585 0x47 と共有。OLED は 0x3C/0x3D が定番（衝突なし）
電源3.3 V 単一（バックライト無し）。MCU 同電源で給電

採用決定 — なぜ ILI9341 2.4"

判定軸ILI9341 採用根拠
デモ動画映え フルカラー RGB565 240×320 で部屋状態を「緑=閉 / 橙=半開 / 赤=全開」と色分け可能。 DigiKey M1 / ROHM EDGE HACK の応募審査で訴求力が段違い。 ProtoPedia ページの 2 分動画にも耐える解像度（OLED 128×64 では 5 タイルが画面いっぱいで限界）
解像度 240×320 = 76,800 px（SH1106 1.3" の 9.4 倍）。 スペクトログラム表示、推論信頼度メーター、複数情報の同時表示が現実的になる
ライブラリエコシステム LVGL v9 公式ドライバ、Adafruit_ILI9341、MCUXpresso SDK の elcdif サンプル — 選択肢が豊富。 SH1106 より圧倒的に「動かしやすい」
SPI 共存性 FC3 SPI を microSD と 同一バスで共有（CS のみ分離）。同時アクセス不可だが、本プロジェクトでは 推論／表示と SD 書込は別フェーズなので問題なし
価格 $13 — SH1106 1.3" ($10) より +$3 だけ。応募審査効果からすれば誤差

採用に伴うトレードオフ

項目ILI9341 で増えるコスト
使用 GPIO 2 → 6 ピン（CS=A2, RESET=A3, DC=A4, BL=A5, MOSI 共有, SCK 共有）。 Arduino ヘッダ A0..A5 を全消費
消費電流 ~10 mA → ~100 mA（バックライト常時点灯）。 電池駆動では BL を PWM 減光してマージン確保
ファーム実装 テスト 1 (main_ili9341_test.c) とテスト 2 (main_lvgl_test.c) を別プロジェクトで作成。 最低構成のドライバ（ili9341.c）は本リポ同梱、 LVGL は MCUXpresso SDK middleware/lvgl からプロジェクトに追加する
筐体寸法 2.4" は 60×40mm 程度 → 3 部屋模型（150×100mm/室）の前面パネルにマウントすれば違和感なし
